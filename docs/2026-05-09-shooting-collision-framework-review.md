# 开拍阶段机制 Review v3：正常片场反应 × 演员入戏偏差

日期：2026-05-09
更新：2026-05-10
状态：已按本方案执行到代码。当前实现使用 `defaultSetReaction` + XML actor 标签作为片场碰撞入口，由 AI 结合 casting、演员现实身份和入组心态自然演化成多句片场过程。

## 2026-05-10 执行说明

本次已落地：

- `scriptSkeleton` 的每个开拍 beat 增加片场压力、默认片场反应、演员改写目标和必须保留功能。
- 片场反应使用 `<actor role="顾沉舟">顾沉舟演员</actor>` 这类 XML 标签标记碰撞入口，玩家不可见。
- AI 生成规则改为：不能只替换一句话，必须把碰撞入口自然展开成 2-5 句连续片场过程。
- 新增 `actor_reaction` 播放类型，用于展示真人演员本人的现场反应，并和剧中角色台词区分。
- 工具干预后，当前幕剩余内容也要生成一小段新的片场过程，而不是只改当前句。

## 核心改法

开拍阶段拆成两层：

1. **剧本骨架提供基础片场反应**
   不是只写“豪门剧情发生了什么”，而是写清楚“一个正常剧组/正常演员拍到这段颠剧情时，会本能产生什么反应”。

2. **AI 生成具体演员碰撞结果**
   AI 不从零发明事故，而是拿“基础片场反应”去和当前演员的入组心态、现实身份、说服结果、片场状态碰撞，生成这一局独有的拍法。

一句话：

> 剧本骨架负责稳定荒唐底色，AI 负责让不同演员把这份荒唐演出不同事故。

## 为什么这样更适合我们

上一版的问题是：演员事故容易变成“人设贴标签”。比如夜场男模就一直夜场，美甲店老板娘就一直情感分析。这样机械，也会削弱整体氛围。

新方案里，喜剧底色先由剧本骨架保证：

- 低成本剧组正在硬拍豪门。
- 剧情本身很颠。
- 所有人都知道这东西很怪。
- 但场地租了，机器开了，时间在烧。
- 所以他们不能真的停下来讨论，只能用草台办法拍下去。

然后 AI 再根据具体演员决定：

- 他会在哪个反应点卡住；
- 他会怎么误读；
- 他会把荒唐剧情理解成什么；
- 老赵要怎么糊过去；
- 玩家工具会把这个碰撞往哪个方向推。

这样演员不是“贴人设”，而是“被这个颠剧本撞出了个人反应”。

## 新的数据结构思路

### 1. 剧本骨架 beat 增加 `setReaction`

每个关键 beat 不只写剧情功能，还写正常片场反应。

```ts
{
  beatId: "act_02_b04",
  beatType: "action",
  speaker: "顾沉舟",
  referenceText: "顾沉舟直接将她拽到面前，当众强吻。",
  mustKeep: true,
  setReaction: {
    whyAbsurd: "这是短剧爽点，但对正常演员来说动作边界非常尴尬。",
    normalActorReaction: "演员会停下来确认是真亲、借位、还是错位。",
    crewPatch: "老赵会要求借位，用镜头贴近和音效制造越线感。",
    lowBudgetPressure: "没有动作指导，也没有时间反复排，场地还在按小时计费。",
    collisionHook: "具体演员会根据自己的边界感、镜头经验、入组心态，把这个动作理解歪。"
  }
}
```

`setReaction` 是骨架提供的稳定底色。它不关心是谁演，只关心“正常人拍到这里会觉得哪里不对劲”。

### 2. 说服后生成 `performanceProfile`

AI 在开拍前为每个演员生成一张入戏偏差卡。

```ts
{
  actorId: "qiu-peng",
  roleId: "zhou-assistant",
  roleName: "周特助 / 顾家长辈",
  roleUnderstanding: "他把周特助理解成一个拼命证明自己还有用的人。",
  shortDramaConfusion: "他不理解为什么助理要替老板跪到这种程度，但觉得短剧可能就是要这么用力。",
  cannotSayNoReason: "他最近太需要一份机会，不敢显得自己事多。",
  personalLeak: "台词碰到求别人留下、顾家不能没有你时，会露出自己的现实情绪。",
  collisionStyle: "先配合，后上头；越荒唐越容易认真。",
  usefulDirection: "导演如果强调职业助理，他会稳定；如果强调真情，他会跑成个人低谷。"
}
```

`performanceProfile` 不直接决定文本，而是参与每个 beat 的碰撞。

### 3. AI 生成每幕时使用碰撞公式

```text
最终拍摄文本
= 剧情 beat
+ setReaction 正常片场反应
+ performanceProfile 演员入戏偏差
+ 当前片场状态
+ 玩家干预历史
```

AI 需要先判断：

1. 这个 beat 本身哪里颠？
2. 正常演员/剧组会怎样反应？
3. 当前演员会把这个反应带向哪里？
4. 老赵如何用低成本办法继续推进？
5. 这一切最后能不能剪进短剧？

## 玩家看到什么

玩家看到的不是后台字段，而是逐句滚动的片场结果。

比如原 beat：

```text
顾沉舟直接将她拽到面前，当众强吻。
```

骨架基础反应：

```text
正常演员会确认借位；剧组会用镜头和音效糊过去。
```

如果演员是“镜头经验强但怕被误读”的女主演员，AI 可能生成：

```text
动作：顾沉舟低头前，沈知意演员先停住，看向导演。
沈知意演员：这段借位吧？正脸拍不到吧？
老赵：借位。正脸不拍，拍他肩膀和你后退。观众会自己脑补。
动作：两个人用非常努力的错位完成了一个看起来很近、实际隔着半个人情世故的吻。
```

如果演员是“过度营业感”的男主演员，AI 可能生成：

```text
动作：顾沉舟演员低头太熟练，熟练到不像失控，像进入了某种服务流程。
沈知意演员往后退了半步，是真的退。
老赵：别那么会。你现在是豪门发疯，不是包厢送客。
```

如果玩家之前用过“导演示范”，AI 可能生成：

```text
动作：顾沉舟演员照着你刚才示范的角度低头，结果连那个尴尬的停顿都一起学了。
老赵：你看，我就说导演不要亲自演。演员会学，学得还挺完整。
```

## 开拍文字稿 v3 方向示例

下面不是完整实现稿，而是每幕的文字方向。完整生成时应由 AI 根据实际演员替换。

### 幕 01｜顾家晚宴外廊羞辱

骨架基础反应：

- 正常人会不理解为什么豪门要在走廊里公开堵前妻。
- 群演不知道自己是亲戚还是宾客，但都知道不能看镜头。
- 老赵的处理方式是：别问身份，站满画面就行。

AI 碰撞方向：

- 沈知意演员会根据自己的入组心态决定是“认真抵抗”还是“先确认镜头”。
- 顾沉舟演员会在“跟我进去”这种霸总命令里暴露自己对豪门的理解。
- 温雅演员会第一次感受到：这戏的体面全靠她自己硬撑。

示例文本：

```text
镜头：顾家晚宴外廊灯火很满，金色门框、假花、租来的水晶灯一起撑出一种很贵但只够拍半天的体面。
动作：群演站在两边，有人还没弄清自己是顾家亲戚还是宾客，只知道不能看镜头。
老赵：都别往镜头里找自己。你们是豪门，不是春晚观众席。
顾家长辈：还知道回来。顾家这些年，倒是第一次为了一个外人这么热闹。
动作：这句一出来，沈知意演员停了半拍，像正常人第一次意识到这家人真的要在走廊里审前妻。
老赵：别停太久。豪门羞辱讲究突然，不能像物业调解。
```

### 幕 02｜顾沉舟公开越线

骨架基础反应：

- 正常演员会确认肢体接触边界。
- 剧组没有动作指导，只能靠“轻拽拍成重拽”。
- 强吻必须借位，否则片场成本和现实风险都不可控。

AI 碰撞方向：

- 顾沉舟演员会根据自己的偏差决定是太熟练、太犹豫，还是太认真。
- 沈知意演员会根据安全感、镜头感、入组心态决定是抗拒、确认、还是直接反制。

示例文本：

```text
动作：顾沉舟上前，一把拽住沈知意手腕。
动作：手刚碰上，两个演员都停了半秒，同时看向导演。
顾沉舟演员：这里真拽吗？
老赵：轻拽。拍出来像重拽。我们主要靠音效。
动作：原剧本该是当众强吻。顾沉舟演员低头到一半，明显慢了。
顾沉舟演员：借位吧？
老赵：当然借位。我们没钱处理工伤，也没钱处理绯闻。
```

### 幕 03｜这一口是你欠顾家的

骨架基础反应：

- “这一口，是你欠顾家的”本身荒唐到演员会想问“这一口是什么口”。
- 正常剧组会避免解释，因为越解释越离谱。
- 老赵会选择直接过，保留短剧疯感。

AI 碰撞方向：

- 顾沉舟演员会在这句台词里暴露他对“深情/占有/羞耻”的理解。
- 沈知意演员会决定是接住荒唐，还是用正常人反应把它打回去。

示例文本：

```text
顾沉舟：这一口，是你欠顾家的。
动作：顾沉舟演员念完这句，空气明显空了一下。
顾沉舟演员：这一口是什么口？
老赵：别问。你一问，我们也要面对这个问题。
沈知意：我欠顾家？你们顾家是有脸说这种话的？
动作：沈知意演员这次没有加情绪，只是平平说出去，反而显得这句更狠。
```

### 幕 04｜温雅失控反击

骨架基础反应：

- 温雅反向强吻/抢回顾沉舟，是短剧疯感升级点。
- 正常演员会确认“刚才到底算不算吻”。
- 道具和服装会制造低成本压力，比如领带不能真扯坏。

AI 碰撞方向：

- 温雅演员会根据自己的入组心态决定她是把这段当雌竞、当委屈、当表演机会，还是当无法理解的疯剧情。
- 顾沉舟演员会暴露自己对“被争夺”的理解。

示例文本：

```text
动作：温雅一步一步走过去，猛地拽住顾沉舟领带。
动作：领带是服装老师临时借的，温雅演员一拽，所有人的视线都先看领带会不会断。
老赵：轻点，那条要还。
温雅：你刚才给她的，我也要。
温雅演员：我刚才真的吻了吗？
老赵：观众觉得你吻了，你就吻了。短剧讲究心理真实。
```

### 幕 05｜顾家选女主人

骨架基础反应：

- “谁能留下谁就是女主人”是封建又荒唐的规则。
- 正常演员会忍不住问这规则谁定的。
- 老赵的答案是：编剧定的，先拍。

AI 碰撞方向：

- 顾家长辈演员会把荒唐规则说成豪门秩序，还是说成临时通知。
- 两位女演员会根据入组心态决定是否真的被“被选”这件事刺激。

示例文本：

```text
顾家长辈：今晚谁能留下，谁就是顾家的女主人。
动作：这句一出来，现场所有人都短暂沉默。
温雅演员：这是什么规则？
老赵：顾家的规则。
温雅演员：顾家谁定的？
老赵：编剧。
沈知意：你们顾家现在连人都要这么选？
```

### 幕 06｜周特助下跪

骨架基础反应：

- 周特助突然下跪是短剧名场面，但正常演员会先考虑地硬不硬、脏不脏、有没有加钱。
- 低成本剧组会把真跪改成半跪，还要拍得像真跪。

AI 碰撞方向：

- 周特助演员会根据入组心态决定：他是怕麻烦、怕丢人、还是越演越真。
- 老赵会在“忠心”和“膝盖别废”之间找剪辑角度。

示例文本：

```text
周特助：少夫人，顾家不能没有您！
动作：原剧本写“扑通一声跪下”。周特助演员看了一眼地面，又看了一眼导演。
周特助演员：真跪？
老赵：半跪。真跪显得我们钱给够了。
动作：周特助半跪下去，姿势非常努力，但看起来像在找掉地上的隐形眼镜。
```

### 幕 07｜只有您和温小姐的嘴

骨架基础反应：

- 这句台词颠到演员和工作人员都会怀疑能不能播。
- 剧组不能停下来审查，因为越审越没法拍。
- 老赵会坚持“先拍，平台的事交给平台”。

AI 碰撞方向：

- 周特助演员会决定是尴尬念完、越念越投入，还是念完自己也后悔。
- 沈知意演员会在这句后代表“正常人”给出反应。

示例文本：

```text
周特助：少夫人，顾总这三年碰过的女人，只有您和温小姐的嘴！
动作：这句说完，现场彻底空白。
周特助演员：这句播得出去吗？
老赵：先拍。播不播是平台的事，拍不拍是我们的事。
沈知意：你们顾家，是真没一个正常人了。
动作：这句说完，没有人觉得她刻薄。大家都觉得她终于说了句正常话。
```

### 幕 08｜沈知意全员审判

骨架基础反应：

- 这是第一集里少数真正有主题表达的段落。
- 正常剧组会突然意识到：前面再土，这里不能完全玩笑化。
- 老赵也会短暂收住，不再只算能不能剪。

AI 碰撞方向：

- 沈知意演员会根据入组心态决定能否接住这段“突然认真”的戏。
- 其他演员会从自己的荒唐状态里被这段话拉住一瞬。

示例文本：

```text
沈知意：一个当众越线，一个当众发疯，一个当众下跪，一个当众选女主人。
动作：她每说一个“当众”，镜头就扫过一个人。扫到周特助时，他还没完全站起来。
老赵：别动，你这个姿势值钱。
沈知意：可你们从头到尾，没有一个人问过——我还想不想踏进这个门。
动作：现场静了两秒。然后隔壁传来装修电钻声。
老赵：停一下。等电钻过去。这句不能被电钻抢。
```

### 幕 09｜真够下贱的

骨架基础反应：

- 终场狠台词要成立，不能被片场喜剧完全打散。
- 低成本质感仍然存在，比如地毯、押金、场务，但不能抢掉女主最后一刀。
- 老赵在这里应该少说，只在收尾时回到草台现实。

AI 碰撞方向：

- 沈知意演员如果前面被带入，最后会有一次真正稳住。
- 顾沉舟演员会根据自己的理解，把“请回去”演成控制、求和、服务承诺或失败挽留。

示例文本：

```text
顾沉舟：那你说。只要你开口，我今天就把整个顾家掀了，也把你请回去。
动作：这次他说“请”字时没有压低声线，像终于意识到前面一直不是请，是拖。
沈知意：顾总。
动作：她只叫了两个字，现场所有人都知道最后一刀要来了。
沈知意：你们顾家求人回去的方式，真够下贱的。
动作：这句没有喊。她说完，连老赵都没有立刻看预算表。
沈知意：那就让他们排队。
动作：沈知意转身离开。裙摆扫过门口那条翘起来的地毯边，差点绊一下，但她稳住了。
老赵：别重来。这个稳住比完美好。
```

## 实现优先级

### 第一阶段：改剧本骨架

给关键 beats 增加 `setReaction`。

先不用大改 UI，也不用改工具逻辑。只要让初始 draft 能稳定拿到“正常人拍颠剧本的反应”。

### 第二阶段：开拍前生成 performanceProfile

说服结束后，把演员、角色、说服结果交给 AI，生成每个演员的入戏偏差卡。

这个阶段决定“具体演员如何和 setReaction 碰撞”。

### 第三阶段：改 draft / revise prompt

要求 AI 生成每幕时必须使用：

- `scriptSkeleton.beats`
- `beat.setReaction`
- `performanceProfiles`
- `actorStates`
- `stats`
- `canonLedger`

并明确要求：

- 每幕 2-4 个碰撞点。
- 碰撞点必须是可见片场反应，不是后台解释。
- 不能机械复述演员身份。
- 短剧剧情功能必须保留。

### 第四阶段：工具影响碰撞方式

工具不只是改剧情，还改变后续碰撞。

- 喊卡：让某个碰撞被压住，但演员可能更紧。
- 改词：改变某个 beat 的荒唐点，后续反应一起变。
- 加鸡腿：降低演员现实阻力，但可能让他更愿意按自己的方式演。
- 导演示范：提供明确示例，但演员可能模仿过度。

## 需要确认

1. `setReaction` 是否应该写进剧本骨架，作为固定创作资产？
2. `performanceProfile` 是否在开拍前生成一次，并作为隐藏状态贯穿整集？
3. 玩家是否需要看到一句轻提示，例如“你不知道他怎么理解了这个角色，但这会影响开拍”？
4. 默认未干预版本的事故密度，是否按“每幕 2-4 个碰撞点”控制？
